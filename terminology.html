Difference between clone and branch

A branch is essentially is a unique set of code changes with a unique name. Cloning a repository pulls down a full copy of all the repository data that GitHub.com has at that point in time, including all versions of every file and folder for the project.

Fork

That a copy of a repository that you manage. For klet you make changes to a project without affectiong the original repository. You can fetch updates from or submit changes to the original repository with pull requests.
